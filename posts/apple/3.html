1 背景

光声成像（Photoacoustic Imaging, PAI）与超声成像（Ultrasound Imaging, US）是两种互补的无创生物医学成像模态。PAI 利用脉冲激光照射生物组织，吸收体（如血红蛋白）因热弹性膨胀产生超声波，由阵列换能器接收并重建为光学吸收分布图像，同时保留了超声的高穿透深度，突破了纯光学成像的散射限制。超声超快平面波成像（Plane-Wave Compounding, PWC）则以单次全孔径发射替代逐线聚焦，通过多角度合成获得接近聚焦质量，帧率可达数千帧每秒，极大拓展了实时成像能力。

两种模态的共同核心是延迟叠加（Delay-and-Sum, DAS）波束合成：将多通道射频（RF）数据按照几何传播时延精确对齐后叠加，重建出每个像素点的声学散射/吸收强度。DAS 的计算复杂度为 O(Nch⋅Npix)O(N_\text{ch} \cdot N_\text{pix})O(Nch​⋅Npix​)，在 128 通道、512×512 像素规模下单帧需约 3.4×1093.4 \times 10^93.4×109 次浮点运算，CPU 串行实现远无法满足实时要求，而 GPU 的大规模cuda并行架构恰好与 DAS 的像素独立性高度匹配。

2 成像物理基础

其中 p(r,t)p(\mathbf{r},t)p(r,t) 为声压，ccc 为声速，β\betaβ 为热膨胀系数，κ\kappaκ 为等温压缩系数，CpC_pCp​ 为定压比热容，H(r,t)H(\mathbf{r},t)H(r,t) 为激光加热函数。

3.1 FIR 高通滤波

滤波器采用 Hamming 窗设计的 100 阶线性相位 FIR 高通滤波器，截止频率由配置文件指定（默认 fc=1 MHzf_c = 1\,\text{MHz}fc​=1MHz，归一化 f^c=fc/fs=0.05\hat{f}_c = f_c / f_s = 0.05f^​c​=fc​/fs​=0.05）。

Hamming 窗系数：

GPU 卷积核将通道维 i、深度维 j、发射角 ii 分别映射到 blockDim.x、blockDim.y、blockIdx.z，三维完全并行：

__global__ void matlabFilterKernel(short** in, int channel, int upPoint,
                                   int line, float* b, int lenB, float* out)
{
    int i  = threadIdx.x + blockIdx.x * blockDim.x; // 通道
    int j  = threadIdx.y + blockIdx.y * blockDim.y; // 采样点
    int ii = blockIdx.z;                             // 发射角

    // 因果卷积（仅用历史样本，保证实时性）
    out[idx] = 0;
    for (int ib = 0; ib < lenB; ++ib)
        if (j - ib >= 0)
            out[idx] += b[ib] * in[ii][(j - ib) * channel + i];
}

注意： 因果卷积在 GPU 上对 j < ib 的边界情况免于原子操作冲突，每个输出点完全独立，无 warp 分歧额外开销。

3.3 亚采样点插值

RF 数据以整数采样索引存储，DAS 计算出的延迟时刻通常落在非整数位置。采用加权幂函数插值：

s(t)≈(1−α)q/2⋅s[⌊t⌋]+αq⋅s[⌊t⌋+1]s(t) \approx (1 - \alpha)^{q/2} \cdot s[\lfloor t \rfloor] + \alpha^q \cdot s[\lfloor t \rfloor + 1] s(t)≈(1−α)q/2⋅s[⌊t⌋]+αq⋅s[⌊t⌋+1]

其中 α=t−⌊t⌋\alpha = t - \lfloor t \rfloorα=t−⌊t⌋ 为小数部分，qqq 为插值幂次（默认 1，即线性插值）。相比线性插值，幂次 q>1q > 1q>1 可提供更强的主瓣抑制，减少旁瓣伪影。

4 光声断层成像 DAS 重建

光声场景使用 128 通道、180° 半圆形阵列（半径 55 mm），阵元坐标在 GPU 常量内存中预存：

static constexpr float kSensorRadius = 55e-3f;

// 初始化：阵元位置存入常量内存（零延迟访问）
__constant__ float c_det_x[kCH];
__constant__ float c_det_y[kCH];

for (int ch = 0; ch < kCH; ++ch) {
    float angle = (float)ch / (kCH - 1) * 179.0f * kPi / 180.0f;
    hx[ch] = kSensorRadius * cosf(angle);
    hy[ch] = kSensorRadius * sinf(angle);
}

DAS 核函数将图像像素点 (ix,iy)(i_x, i_y)(ix​,iy​) 映射为一个 CUDA 线程，对全部 128 个通道累加：

__global__ void k_das_f32(const float* rf_ch, float* out_f,
    int n, float img_w, float c, float fs, int up)
{
    int ix = blockIdx.x * blockDim.x + threadIdx.x;
    int iy = blockIdx.y * blockDim.y + threadIdx.y;

    float px = -img_w * 0.5f + ix * (img_w / (n - 1));
    float py = -img_w * 0.5f + iy * (img_w / (n - 1));

    float das_sum = 0.0f;
    for (int ch = 0; ch < kCH; ++ch) {
        float dist = hypotf(px - c_det_x[ch], py - c_det_y[ch]);
        int it0 = (int)roundf(dist / c * fs);
        if (it0 >= 0 && it0 < up)
            das_sum += rf_ch[ch * up + it0];
    }
    out_f[iy * n + ix] = das_sum;
}

常量内存的带宽优势在此充分体现：c_det_x[ch] 对同一 warp 内全部线程广播，等效于寄存器访问延迟（~1 周期），相比全局内存节省约 100 倍带宽开销。

5.2 显存布局与访问模式

RF 数据布局为 [angle][channel][sample]（行主序），DAS 核访问模式为：

  • 内层循环（通道维）：rf[angle * CH * UP + ch * UP + t]，跨步访问，每次跨 UP = 8192 个 float
  • 优化方向：转置为 [angle][sample][channel] 布局后，warp 内相邻线程访问相邻通道，可实现合并访问（coalesced access），预期带宽利用率提升 4～8 倍

5.3 显存复用策略

系统采用懒初始化 + 尺寸缓存策略，避免每帧重新 cudaMalloc（显存分配耗时约 100 μs）：

void initGPU(int channel, int upPoint, int line, ...) {

6 频谱熵自动声速对焦

声速估计是光声和超声重建中的关键问题。错误的声速导致延迟计算偏差，使图像出现散焦模糊。本系统实现了基于频谱熵最小化的两级自动对焦算法。

6.1 频谱熵原理

对已重建的 B 模式图像 I(x,z)I(x,z)I(x,z) 做二维 DFT，得到频谱幅度 ∣F(u,v)∣|F(u,v)|∣F(u,v)∣。定义归一化概率分布：

p(u,v)=∣F(u,v)∣∑u,v∣F(u,v)∣p(u, v) = \frac{|F(u, v)|}{\sum_{u,v} |F(u, v)|} p(u,v)=∑u,v​∣F(u,v)∣∣F(u,v)∣​

频谱熵（Shannon 熵）：

H(c)=−∑u,vp(u,v)ln⁡p(u,v)\mathcal{H}(c) = -\sum_{u,v} p(u,v) \ln p(u,v) H(c)=−u,v∑​p(u,v)lnp(u,v)

物理直觉：声速正确时，点目标 PSF 最紧致，图像最锐利，频谱能量集中（熵最小）；声速偏差导致散焦，能量弥散（熵增大）。因此 c^=arg⁡min⁡cH(c)\hat{c} = \arg\min_c \mathcal{H}(c)c^=argminc​H(c)。

__global__ void hilbertFilterKernel(cufftComplex* in, int channel,
                                    int upPoint, cufftComplex* out)
{
    // 正频率分量乘 2，负频率置零
    for (int i = 1; i < upPoint / 2 + upPoint % 2; ++i) {
        out[i + j * upPoint].x = pin[i].x * 2.0f;
        out[i + j * upPoint].y = pin[i].y * 2.0f;
    }
    for (int i = upPoint / 2 + 1; i < upPoint; ++i) {
        out[i + j * upPoint] = {0.0f, 0.0f};
    }
}

// hilbertAbsKernel：计算包络 + 对数压缩
__global__ void hilbertAbsKernel(cufftComplex* in, ..., float logBase, float* out)
{
    float amp = sqrtf((in[id].x / upPoint) * (in[id].x / upPoint)
                     + (in[id].y / upPoint) * (in[id].y / upPoint));
    out[id] = log10f(amp) / logBase; // 可调对数底数
}

7.2 TGC 时间增益补偿

超声波在传播过程中因衰减系数 μ\muμ 导致深层信号减弱：

A(z)=A0⋅e−μzA(z) = A_0 \cdot e^{-\mu z} A(z)=A0​⋅e−μz

TGC 通过对不同深度施加指数增益予以补偿，将深度范围分为 9 段，每段增益 gi∈[0,100]g_i \in [0, 100]gi​∈[0,100]：

// 在 resizeAndCropKernel 中，按输出行 jj 插值 TGC 增益
int tid  = jj / 64;
float a0 = expf((100 - tgcs[tid]     - 50) / 50.0f * 2.0f);
float a1 = expf((100 - tgcs[tid + 1] - 50) / 50.0f * 2.0f);
float k  = (a1 - a0) / 64.0f;
t *= ((jj - tid * 64) * k + a0);

7.3 动态范围压缩（对数压缩 + Gamma 校正）

B 模式显示通常需将 40～60 dB 动态范围映射至 8 bit 灰度，采用两段策略：

标准模式（scaleduse = 0）：线性归一化至 [0,255][0, 255][0,255]

I_\text{out}(x,z) = \text{clip}\!\left(\frac{I(x,z) - I_\min}{I_\max - I_\min}, 0, 1\right) \times 255

增强模式（scaleduse = 1）：对数压缩 + 百分位归一化 + Gamma 校正

IdB(x,z)=20log⁡10∣I(x,z)∣+GI_\text{dB}(x,z) = 20 \log_{10}|I(x,z)| + G IdB​(x,z)=20log10​∣I(x,z)∣+G

t=clip ⁣(IdB−P5P95−P5,0,1)γt = \text{clip}\!\left(\frac{I_\text{dB} - P_5}{P_{95} - P_5}, 0, 1\right)^{\gamma} t=clip(P95​−P5​IdB​−P5​​,0,1)γ

Iout=round(10+t×240)I_\text{out} = \text{round}(10 + t \times 240) Iout​=round(10+t×240)

其中 P95P_{95}P95​ 为图像 dB 值的 95 百分位，有效抑制高亮伪影；γ=gamma_int×0.1\gamma = \text{gamma\_int} \times 0.1γ=gamma_int×0.1 提供灰度对比度调节。

9 性能与优化展望

9.1 当前瓶颈分析

FIR 滤波 (时域 100 阶)       │████████████████░░░░░│ ~35%
DAS 重建 (7 角 × CH × UP)   │█████████████████████│ ~45%
Hilbert + Resize           │████░░░░░░░░░░░░░░░░░│ ~10%
cudaMemcpy (H↔D)           │███░░░░░░░░░░░░░░░░░░│ ~10%

DAS 核的主要瓶颈在于对 rf_ch[ch * UP + t] 的非合并访问（通道间跨步 UP = 8192 个 float = 32 KB），建议转置为 [sample][channel] 布局以实现 warp 级合并访问。

9.2 优化路线图

  1. 共享内存缓存：在 DAS 核内将每通道的 RF 数据片段预加载到共享内存，减少全局内存访问次数

  2. CUDA Streams 流水线：将帧传输（cudaMemcpyAsync）与前一帧计算重叠，提升吞吐量

  3. cuBLAS 加速角度求和：将多角度 DAS 结果的累加表示为矩阵加法，借助 cuBLAS 的优化 GEMM 路径

  4. 半精度（FP16）加速：对 DAS 求和使用 FP16 累加，RTX 系列 GPU 的 FP16 Tensor Core 吞吐量是 FP32 的 8 倍，可在保持足够精度的前提下大幅提速

  5. 自适应声速缓存：将自动对焦估计的最优声速跨帧平滑，利用时序相关性减少搜索范围，从而将自动对焦耗时从 100 ms 降至 20 ms 以内

10 总结

本文以实际工程代码为基础，系统梳理了 CUDA 加速超声平面波成像与光声断层成像的完整技术链路：

  • 物理层：声波方程 → 时延公式 → 动态孔径约束，为重建算法提供理论依据
  • 信号层：Hamming 窗 FIR 高通滤波（时域）/ IIR 带通频域滤波（cuFFT），保留有效声学频带
  • 重建层：多角度 DAS 波束合成，利用 CUDA 三维线程块将通道、深度、角度三维并行化
  • 后处理层：cuFFT Hilbert 变换解析信号包络，TGC 深度增益补偿，对数压缩 + Gamma 校正映射至灰度图像
  • 智能层：基于频谱熵最小化的两级粗精搜索声速自动对焦

GPU 并行架构与 DAS 算法的天然匹配，使实时超声/光声成像在消费级 GPU 上成为可能，为血管弹性成像、心脏功能成像等高帧率应用奠定了基础。
